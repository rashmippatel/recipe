Data Model (Prisma)
model Recipe {
  id            String   @id @default(cuid())
  name          String
  isVeg         Boolean  @default(true)
  cookTimeMin   Int
  mealTypes     String[] // ["Breakfast","Lunch","Evening Snack","Dinner"]
  instructions  String
  sourceType    String   // "website" | "youtube" | "book" | "other"
  sourceUrl     String?  // nullable
  famous        Boolean  @default(false) // for Workflow 1 toggle
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  ingredients   IngredientOnRecipe[]
}

model Ingredient {
  id    String  @id @default(cuid())
  name  String  @unique
  unit  String? // "g" | "ml" | "pcs" | null
  recipes IngredientOnRecipe[]
}

model IngredientOnRecipe {
  id          String   @id @default(cuid())
  recipeId    String
  ingredientId String
  quantity    Float    // required amount for the recipe (in Ingredient.unit if present)
  recipe      Recipe   @relation(fields: [recipeId], references: [id])
  ingredient  Ingredient @relation(fields: [ingredientId], references: [id])

  @@unique([recipeId, ingredientId])
}

Matching Algorithm (Workflow 1)

Implement a deterministic scoring function on the server:

Filters (hard constraints):

If vegOnly=true then exclude recipes with isVeg=false.

If meal provided, only include recipes where mealTypes includes that meal.

If maxTime provided, only include recipes where cookTimeMin <= maxTime.

If includeFamous=false, set candidatePool = all recipes where famous=false; else all recipes.

Ingredient availability & quantity match:

Input: array of {ingredientName, qtyAvailable}. Normalize names (case-insensitive).

For each recipe:

Let R = required ingredients set.

Let A = available ingredients set (user input).

Compute:

coverage = |R ∩ A| / |R| (0..1).

quantitySatisfaction: for each ingredient in R ∩ A, contribute 1 if qtyAvailable >= recipe.requiredQty else contribute qtyAvailable / requiredQty. Average over R. (If ingredient not available, contributes 0.)

score = 0.60 * coverage + 0.30 * quantitySatisfaction + 0.10 * freshnessBoost, where:

freshnessBoost = 0.1 if recipe’s cookTimeMin <= maxTime/2 (encourages quicker options), else 0.

Exact match prioritization: If coverage === 1 and quantitySatisfaction === 1, add +0.25 bonus.

Sort by score desc; return top 3 with a breakdown: coverage, quantitySatisfaction, bonusApplied, finalScore.

Document the formula in README.md.
